# 协作学习仿真

## 定义

LLM 多智能体系统模拟**协作学习过程**，用于研究学习脚手架（scaffolding）策略的效果，是教育 AI 研究的新兴方法论。

## 仿真测试框架的新进展（arXiv:2604.11161）

"Simulation-Based Method for Testing Collaborative Learning Scaffolding Using LLM Multi-Agent Systems" 提供了更系统的仿真测试方法论：

### 仿真流程
1. **定义脚手架策略** — 研究者精确描述脚手架规则
2. **智能体初始化** — 分配角色（Leader, Supporter, Expounder, Rebutter, Summarizer）
3. **运行仿真** — MetaGPT/GPT-4o 执行多轮协作
4. **结果评估** — 对比有/无脚手架的协作质量差异

### 评估指标
| 指标 | 无脚手架 | 有脚手架 |
|------|---------|---------|
| 话语多样性 | 低 | 显著提升 |
| 交互深度 | 浅层 | 深层推进 |
| 内容重复率 | 高 | 明显降低 |
| ICAP 层级 | P/A 层为主 | C/I 层出现 |

### 关键洞察
- **脚手架的作用是创造认知冲突** — 强制智能体在发言前深度思考
- **"Deep Think before Speak"** 策略验证有效：促使 Supporter/Rebutter 提供更深层反馈
- 仿真结果与真实教室研究结论一致 → **仿真方法论已具备外部效度**

利用 LLM 多智能体系统（Multi-Agent）模拟学习者之间的协作交互：
- 每个智能体代表一个学习者角色
- 智能体之间进行讨论、辩论、互助
- 研究者可以控制变量（脚手架策略、学习者特征等）

## 验证框架

| 平台 | 模型 | 验证方式 |
|------|------|---------|
| MetaGPT + GPT-4o | 多智能体协作 | 脚手架策略效果验证 |

## 核心价值

- **成本低**：无需招募真实学习者，可快速迭代
- **可控性强**：变量可精确操控
- **可解释**：可记录完整的协作对话过程

## 局限

- 真实性受限于 LLM 本身的社交/认知能力
- 不能完全替代真实教室观察
