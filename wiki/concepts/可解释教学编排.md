# 可解释教学编排

## 定义

可解释教学编排（Interpretable Pedagogical Orchestration）指在多智能体教育系统中，将原本作为黑盒的单一大型语言模型替换为**多个专业化特化模型组合**，每个组件负责特定教学功能，通过显式协同实现自适应教学。其核心目标是在保持个性化教学能力的同时，让教学决策过程完全可审计、可解释。

## 问题背景：黑盒 LLM 的局限

当前教育对话系统多使用单一 LLM（GPT-4 等），其内部决策机制对教师和学习者完全不透明：
- 无法解释"为什么系统选择这样回应"
- 无法追踪"哪个教学策略正在生效"
- 无法让教师审核或干预系统的教学决策

## 核心架构：特化模型集成

AIED 2026 论文 "From Untamed Black Box to Interpretable Pedagogical Orchestration" 提出的框架：

| 特化组件 | 职责 |
|----------|------|
| **诊断智能体** | 评估学习者当前状态、知识缺口 |
| **策略选择智能体** | 根据诊断结果选择教学策略 |
| **对话生成智能体** | 生成符合所选策略的教学回复 |
| **评估智能体** | 事后评估本次教学交互的质量 |

每个智能体的决策都有**显式理由输出**，教师可逐一审核。

## 关键设计原则

1. **职责单一化** — 每个特化模型只负责一个功能，降低复杂性
2. **决策可追溯** — 每个教学选择附带理由文本，而非隐式推理
3. **教师在环** — 关键决策节点需教师确认，系统不自主执行高风险教学行为
4. **可组合替换** — 组件可独立升级，不影响整体架构

## 与传统单体 LLM 的对比

| 维度 | 单体 LLM | 可解释编排架构 |
|------|---------|--------------|
| 决策可解释性 | 低（隐式推理） | 高（显式理由链） |
| 教师审核能力 | 差 | 强（每个节点可审计） |
| 系统可靠性 | 不可控 | 可逐步验证 |
| 定制化能力 | 受限于 prompt 工程 | 组件级别独立优化 |
