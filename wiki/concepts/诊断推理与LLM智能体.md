# 诊断推理与LLM智能体

## 定义

诊断推理（Diagnostic Reasoning）是医学、教育等领域中通过系统分析症状/表现来推断潜在原因的高阶认知过程。在 AI 教育中，LLM 智能体正被用于**模拟和辅助诊断推理学习**，帮助学生发展从现象到本质的分析能力。

## 核心问题：结构化 vs 提问式脚手架

LAK 2026 论文 "Structuring versus Problematizing"（Güreş et al., 2026）系统对比了两种 LLM 脚手架策略：

### 结构化脚手架（Structuring）
- 为学生提供**预先设计好的分析框架**
- 引导学生按固定步骤完成诊断
- 优点：降低认知负荷，新手友好
- 缺点：可能限制自主推理的深度

### 提问式脚手架（Problematizing）
- 通过**苏格拉底式提问**引导，而非直接给框架
- 鼓励学生自主建构诊断路径
- 优点：促进深层理解和元认知
- 缺点：对新手可能过于开放，导致挫败感

### 关键发现

| 发现 | 含义 |
|------|------|
| 问题解决式脚手架提升**第一次诊断准确性** | 结构化框架更有效 |
| 结构化脚手架促进**推理过程完整性** | 学生覆盖更多诊断步骤 |
| 两种策略对**最终诊断准确性**差异不显著 | 殊途同归 |
| 学生对两种策略的主观体验差异显著 | 需考虑学习者偏好 |

## 从正确性到理解力

"From Correctness to Comprehension: AI Agents for Personalized Error Diagnosis in Education"（Zhang et al., 2025）提出：

**传统错误纠正** → 告诉学生"错了，正确答案是X"
**诊断推理教育** → 引导学生发现"为什么错，哪里理解的偏差"

后者不仅纠正当前错误，更建立对未来类似问题的预防能力。

## LLM 智能体在诊断推理中的角色

| 角色 | 描述 |
|------|------|
| **诊断者（Diagnostician）** | 分析学生表现，识别知识缺口（参考 GraphMASAL 的三智能体架构） |
| **问题生成器** | 生成针对诊断结果的定制化练习 |
| **元认知导师** | 引导学生反思自己的推理过程 |
| **反馈生成器** | 提供解释性反馈，而非仅判断对错 |

## 设计启示

1. **新手先用结构化** — 积累足够案例后引入提问式
2. **诊断过程要外显** — 让学生看到推理步骤，而非只看结果
3. **错误是学习资源** — 刻意设计诊断错误原因的训练
4. **教师始终在环** — 高风险诊断决策需教师确认
